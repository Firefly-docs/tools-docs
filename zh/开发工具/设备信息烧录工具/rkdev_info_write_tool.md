# RKDevInfoWriteTool

RKDevInfoWriteTool 是 Rockchip 平台的设备信息烧录工具，可向设备的 VendorStorage 写入 SN、WIFI MAC、LAN MAC 和 BT MAC 等设备信息。本章节参考只介绍 SN 和 MAC 地址的读取、写入和连续自增烧写。

<font color=red>**注意以下操作均在 Windows PC 端操作，烧写时建议每次只连接一台待烧录设备。**</font>

## 准备工作

1. 工具：RKDevInfoWriteTool.zip (your_rockchip_sdk_path/tools/windows/RKDevInfoWriteTool.zip)。

2. 解压工具包，保持目录结构不变：

    ```
    RKDevInfoWriteTool-v1.3.7
        ├── RKDevInfoWriteTool.exe
        ├── adb.exe
        ├── RKUpgrade.dll
        ├── config.ini
        ├── Language
        └── plugs
    ```

3. 将设备连接到 Windows PC，并让设备进入 Loader 或 Maskrom 模式。

    - Loader 模式：设备已有可启动固件时使用。
    - Maskrom 模式：空片或无法进入 Loader 时使用，通常需要在工具中选择正确的 `MiniLoaderAll.bin`。

4. 双击运行 `RKDevInfoWriteTool.exe`。工具识别到设备后，底部状态栏会显示已发现设备。

## 主界面

主界面可选择 `SN`、`WIFI`、`LAN`、`BT` 等写入项，并执行读取或写入操作：

![RKDevInfoWriteTool 主界面](../../../img/rkdev_info_write_tool_main.png)

主界面常用选项如下：

| 选项 | 说明 |
| --- | --- |
| `单次读写` | 勾选后，每次接入设备都需要手动点击 `读取` 或 `写入`。 |
| 连续读写 | 不勾选 `单次读写` 时，接入新设备会自动触发读取或写入操作，适合批量烧写。 |
| `读取` | 读取已勾选项目的当前值，用于写入后校验。 |
| `写入` | 将已勾选项目的当前配置值写入设备。 |
| `LOADER` | 设备处于 Maskrom 模式时，选择正确的 MiniLoader 文件。 |

## 写号配置

点击主界面菜单栏 `设置`，进入写号配置界面：

![RKDevInfoWriteTool 写号配置](../../../img/rkdev_info_write_tool_settings.png)

常用配置如下：

| 配置 | 说明 |
| --- | --- |
| `手动` | 在主界面手动输入 SN 或 MAC 地址，也可配合扫描枪输入。 |
| `自增` | 设置起始值后，每次写入成功自动将数字部分加 1，适合连续烧写。 |
| `文件` | 从文本文件逐行读取 SN 或 MAC 地址，每写入一台设备消耗一行。 |
| `强制写入` | 勾选后即使设备已有对应信息，也会重新写入。未勾选时，已写入过的项目可能被跳过。 |
| `写后重启` | 写入完成后自动重启设备。 |
| `兼容模式` | 旧版本系统或特殊场景使用。一般新系统优先不勾选。 |

## 单台设备写入

### 写入 SN

1. 在主界面勾选 `SN`。

2. 如需手动输入，在 `设置` 中切换到 `SN` 页签，将取号模式设置为 `手动`，保存配置。

3. 在主界面 `SN` 输入框中填写要写入的序列号。

    示例：

    ```
    RK2026070801
    ```

4. 点击 `写入`。写入完成后点击 `读取`，确认读取到的 `SN` 与写入值一致。

### 写入 MAC 地址

工具界面中的 MAC 地址项如下：

| 项目 | 说明 |
| --- | --- |
| `WIFI` | WIFI MAC 地址 |
| `LAN` | 以太网 MAC 地址 |
| `BT` | 蓝牙 MAC 地址 |

1. 根据需要勾选 `WIFI`、`LAN` 或 `BT`。

2. 在对应输入框中填写 MAC 地址。MAC 地址使用 12 位十六进制字符，不需要冒号或横线分隔。

    示例：

    ```
    001A2B3C4D5E
    ```

    如果已有 MAC 地址格式为 `00:1A:2B:3C:4D:5E`，填写时请去掉冒号。

3. 点击 `写入`。写入完成后点击 `读取`，确认读取到的 MAC 地址与写入值一致。

## 连续自增烧写

连续自增烧写适合批量生产：工具每成功写入一台设备，就自动将配置项中的数字部分加 1，然后用于下一台设备。

### SN 连续自增

1. 打开 `设置`，选择 `SN` 页签。

2. 勾选 `选择`，将取号模式设置为 `自增`。

3. 在 `起始` 中填写第一个 SN，例如：

    ```
    RK2026070801
    ```

4. 如 SN 中有固定前缀或后缀，在 `前缀`、`后缀` 中填写固定部分。例如 `RK2026070801` 可将 `RK` 作为前缀，使后面的数字自增。

5. 设置 `计数`，表示可连续写入的数量。

6. 如需按十六进制递增，勾选 `十六进制`；否则默认按十进制递增。

7. 保存配置，返回主界面。

8. 取消勾选主界面 `单次读写`，连接第一台设备后点击 `写入`。写入成功后，断开当前设备并接入下一台设备，工具会自动使用下一个 SN 继续写入。

### MAC 连续自增

LAN MAC 配置界面如下图所示。如需配置多个 LAN MAC，可勾选 `多LANMAC` 后在 `MAC0`、`MAC1`、`MAC2`、`MAC3` 中配置对应地址：

![RKDevInfoWriteTool LAN MAC 配置](../../../img/rkdev_info_write_tool_manual_mode.png)

1. 打开 `设置`，选择需要配置的 `WIFI MAC`、`LAN MAC` 或 `BT MAC` 页签。

2. 勾选 `选择`，将取号模式设置为 `自增`。

3. 在 `起始` 中填写第一个 MAC 地址，例如：

    ```
    001A2B3C4D5E
    ```

4. 设置 `计数`。如果需要十六进制递增，勾选 `十六进制`。

5. 保存配置，返回主界面，取消勾选 `单次读写`。

6. 点击 `写入` 烧写第一台设备。后续每接入一台新设备，工具会自动写入自增后的 MAC 地址。

## 注意事项

- SN 和 MAC 地址应保证唯一，避免多台设备使用相同值。
- PDF 中说明 SN 默认要求符合 GMS 规范：长度不大于 14，且为字母开头的数字字母组合。不符合规范时，设备重启后可能显示系统随机生成的 SN。
- MAC 地址必须为 12 位十六进制字符，可使用数字 `0-9` 和字母 `A-F`。
- 写入前确认只勾选本次需要烧录的项目，避免误写其它设备信息。
- 使用连续读写前，建议先用一台设备验证写入和读取结果，再开始批量烧写。
- 如工具未识别设备，请检查 USB 连接、驱动安装情况，以及设备是否处于 Loader 或 Maskrom 模式。
